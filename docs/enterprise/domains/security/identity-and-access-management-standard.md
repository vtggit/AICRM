# Identity and Access Management Standard

- Authenticate users through a centralized enterprise identity provider.
- Avoid local username/password stores for new enterprise applications.
- Implement role-based access control by default.
- Prefer group-to-role mapping from the central identity platform where practical.
- Use OIDC for modern application authentication by default.
- Use the application authentication standard to choose between web sessions, JWT bearer tokens, and service identities.
